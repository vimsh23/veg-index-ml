# 🌿 Vegetation Index ML — Remote Sensing Tool

Random Forest based vegetation index computation and land cover classification.
Supports NDVI, EVI, SAVI, NDWI, MSAVI from GeoTIFF or CSV inputs.

## 🚀 Quick Start (Docker — recommended)

```bash
# 1. Clone / unzip the project
cd veg-index-ml

# 2. Train the model first (runs inside Python, not Docker)
cd backend
pip install -r requirements.txt
python sample_data/generate_sample_data.py   # creates sample_data/training_data.csv
python model/train.py                        # creates model/rf_model.joblib
cd ..

# 3. Start everything with Docker Compose
docker-compose up --build

# Frontend → http://localhost:3000
# API Docs → http://localhost:8000/docs
```

## 🔬 Training with Your Own Data

Your CSV must have these columns (band reflectance values, 0.0–1.0 range):

| Column | Description             |
|--------|-------------------------|
| blue   | Blue band (B2)          |
| green  | Green band (B3)         |
| red    | Red band (B4)           |
| nir    | Near-infrared (B8/B8A)  |
| class  | Integer land cover label|

> If your values are in DN (0–10000 range), the script auto-scales them.

```bash
cd backend
python model/train.py \
  --data   path/to/your_data.csv \
  --target class \
  --output model/rf_model.joblib \
  --estimators 300 \
  --cv-folds 5
```

Training output includes:
- Per-class accuracy (classification report)
- 5-fold cross-validation score
- Feature importance bar chart in terminal
- Saved model at `model/rf_model.joblib`

## 🌐 API Endpoints

| Method | Endpoint                     | Description                          |
|--------|------------------------------|--------------------------------------|
| GET    | `/`                          | Health + welcome                     |
| GET    | `/health`                    | Model loaded status                  |
| GET    | `/indices`                   | List available indices               |
| POST   | `/compute-index/csv`         | Compute index from CSV               |
| POST   | `/compute-index/geotiff`     | Compute index from GeoTIFF           |
| POST   | `/compute-all/csv`           | Compute all indices from CSV         |
| POST   | `/predict/csv`               | Predict land cover from CSV          |

**Example: Compute NDVI from CSV**
```bash
curl -X POST "http://localhost:8000/compute-index/csv?index=NDVI" \
     -F "file=@sample_data/training_data.csv"
```

**Interactive docs**: http://localhost:8000/docs

## ☁️ Deploy to Cloud (Free Tier)

### Backend → Render

1. Push to GitHub
2. Go to [render.com](https://render.com) → New → Blueprint
3. Connect repo → Render reads `render.yaml` automatically
4. After deploy, copy your Render URL (e.g. `https://veg-ml-api.onrender.com`)

> **Important**: After training locally, upload `rf_model.joblib` to the Render disk
> via the Render dashboard → Disk → Files, or via `rsync`/`scp`.

### Frontend → Vercel

1. Push to GitHub (same repo)
2. Go to [vercel.com](https://vercel.com) → New Project → Import repo
3. Set root directory to `frontend/`
4. Deploy — Vercel auto-reads `vercel.json`
5. In the UI, update the API URL field to your Render backend URL

## 📊 Vegetation Indices Reference

| Index | Formula                              | Use Case                  |
|-------|--------------------------------------|---------------------------|
| NDVI  | (NIR−Red)/(NIR+Red)                  | Vegetation density        |
| EVI   | 2.5*(NIR−Red)/(NIR+6*Red−7.5*Blue+1)| Dense canopy, atmospheric |
| SAVI  | (NIR−Red)/(NIR+Red+0.5)*1.5         | Sparse vegetation / soil  |
| NDWI  | (Green−NIR)/(Green+NIR)             | Water bodies detection    |
| MSAVI | (2*NIR+1−√((2*NIR+1)²−8*(NIR−Red)))/2 | Bare soil areas        |
